### 6.3 iLink数据包格式

**LongLink数据包（TCP）**：
```
+--------+--------+--------+--------+
| total_length (4B, Big-Endian)     |  包总长
+--------+--------+--------+--------+
| cmdid (2B)                        |  命令ID (校验值0x1011)
+--------+--------+--------+--------+
| seq (4B, Big-Endian)              |  序列号
+--------+--------+--------+--------+
| body_length (4B, Big-Endian)      |  body长度
+--------+--------+--------+--------+
| body (变长)                       |  业务数据(protobuf)
+--------+--------+--------+--------+
最大包长: 1MB (0x100000)
```

**iLink2数据包结构**：
```
偏移 0:   1字节 - 版本/类型
偏移 4:   1字节 - 字段 (<=0x17)
偏移 5:   1字节 - 小时 (tm_hour)
偏移 8:   4字节 - 长度 (含payload)
偏移 80:  1字节 - 标志
偏移 84:  4字节 - CRC32校验
偏移 88:  1字节 - 标志 (校验时需==0xFF)
payload:  变长 - 实际数据 (8字节对齐填充)
```

**CSPkg封装格式**：
```
[4字节长度(小端序)][CSPkg TDR序列化数据]

CSPkg
├── CSPkgHead
│   ├── dwMsgID: UInt32     ← 命令ID(CSCMD_ID_DEF)
│   ├── iVersion: Int32     ← 版本号
│   ├── dwSvrPkgSeq: UInt32 ← 服务端包序号
│   └── dwReserve: UInt32   ← 保留字段
└── CSPkgBody
    ├── dataObject: ProtocolObject  ← selector路由
    ├── szData: Byte[]              ← 原始数据
    └── selector2TypeClassId        ← dwMsgID→消息类型映射
```

### 6.4 MTK认证数据包

```
固定264字节: 4字节客户ID + 4字节魔数(0x534D41) + 256字节加密数据 (小端序)
```

### 6.5 iLink加密算法枚举

